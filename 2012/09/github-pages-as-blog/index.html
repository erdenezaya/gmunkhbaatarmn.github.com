Өмнө нь зочилж байсан уншигчид блог маань шинэчлэгдсэн байгааг анзаарсан байлгүй. Блог маань blogger (blogspot) платформ дээр байрлаж байсан бол одооноос github хуудас дээр байрлах боллоо.

Github хуудасны тухай

Github хуудас нь энгийнээр хэлбэл статик веб хостингийн үйлчилгээ юм. Github дээр бүртгэлтэй хэн бүхэн энэ үйлчилгээг ашиглаж болох ба, өөрийн ник нэр бүхий дэд домайн дээр эзэмших эрхтэй юм. Өөрөөр хэлбэл таны github хэрэглэгчийн нэр "foo" бол таны github хуудас "foo.github.com" хаяг дээрээс ажиллана гэсэн үг.

Github хуудастай болох маш амархан бөгөөд та github дээр <username>.github.com гэсэн repo нээснээр github хуудастай болно. Мэдээж энэ repo доторх бүх файлууд <username>.github.com хаягаар дуудагдах болно. Бас нэг зүйл дурдахад <username>.github.com гэж хаяг дуудах нь <username>.github.com/index.html гэж файлыг дуудахтай адилаар ажиллах тул та repo дотроо index.html файл байлгах хэрэгтэй.

Github хуудсанд хувийн домайн холбох

Хэрвээ та таамаглаж амжсан бол таны зөв. Блог маань github хуудас дээр байрлаж байгаа хэрнээ itblog.gmb.mn хаягтай байна шүү дээ. Тэгэхлээр github хуудсандаа домайн холбож болдог байх нь.

Маш амархан. <username>.github.com repo дотроо CNAME нэртэй файл үүсгээд дотор нь домайн нэрээ бичнэ. Дараа нь домайн сэрвэр дээрээ жижигхэн тохиргоо оруулна. Дэд домайн бол CNAME бичилт, домайн бол A бичилт оруулна. Домайн бичилтийн IP хаягийг github help-ээс лавлаж хараарай.

За тэгэхээр блог маань github.com/gmunkhbaatarmn/gmunkhbaatarmn.github.com хаяг дээр байрлаж байгаа гэсэн үг төдийгүй та бүх файлуудыг харах эрхтэй, git ашиглан clone, fork хийх, pull request явуулах эрхтэй гэсэн үг юм :)

Блог статик байж болох уу?

За одоо ингээд статик хостингтой болчихдог юм байж, яаж үүнийг блог болгож ашиглах вэ? Блог статик байж болох уу?

Хэрвээ сайн анзаарах юм бол блог бол маш энгийн бүтэцтэй веб сайт юм. Блог гэдэгт нүүр хуудас буюу бичлэгүүдийн жагсаалт, бичлэг харуулдаг хуудас, сэтгэгдэл бичих хэсэг, бичлэгүүдийг ангилалаар шүүж харуулдаг бас нэг хуудас байхад хангалттай байдаг. Блогийн програмууд эдгээр хуудсуудыг динамик байдлаар боловсруулж харуулдаг. Гэвч эдгээр хуудсууд байнга шинэчлэгддэггүй, зөвхөн бичлэг оруулах үед л шинэчлэгддэг. Тийм бол эдгээр хуудсуудыг харах болгондоо боловсруулалгүй, нэг боловсруулж аваад кэйшлэх, хэзээ нэгэн цагт бичлэг оронгуут кэйшээ шинэчилж болмоор санагдаж байгаа биз? Тэгээд бүүр давраад сайтаа тэр чигээр нь кэйшлэчихвэл ямар уу? :)

Яг ийм маягаар статик хуудсаа блог байдлаар ашиглаж болох нь ээ. Статик хуудсанд маань блогийн хамгийн сүүлийн хувилбар бүхий кэйш ажиллаж, зөвхөн бичлэг орох үед кэйш буюу блог маань шинэчлэгдэж байх нь байна.

  • Байз! Сэтгэгдэл оруулах үед яах вэ? Үүнийг бас кэйшлэх үү?

За юу л даа... Хмм... энэ удаад мэдээж жаахан булхайцах хэрэг гарна. Гадны сэтгэгдэл авдаг үйлчилгээнүүдээс embed код аваад ашиглаж болно :D Энэ блог дээр жишээ нь disqus холбосон байгаа.

Статик блог үүсгэгчүүд

Статик блог үүсгэгч төрөл бүрийн програмууд гарчээ. Би Jekyll (Ruby) vs Hyde (Python) гээд хоёр програмыг удаан харьцуулсан боловч эцэст нь нэрнээс нь болоод Jekyll-г сонгосон :).

Та ямар ч програм сонгосон бай бүгд л эцэстээ статик веб үүсгэх тул санаа амар github хуудас дээрээ байрлуулж болох юм. Доор нэмэлт холбоосууд хэсэгт статик блог үүсгэгч сонгоход тань хэрэг болох линкүүдийг орууллаа.

Яагаад заавал статик блог гэж?

Төрөл бүрийн үнэгүй динамик блог програмууд байсаар байтал яагаад заавал блогоо статик болгож өөртөө ажил удах хэрэгтэй гэж?

  • Блогийн хуулбар тань компьютер дээр чинь байрлана, та интернетгүй үед ч гэсэн блог руугаа хандана, блогоосоо юм лавлаж харахад хялбар байна.
  • Блогийнхоо бичлэгийг хүссэн форматаараа бичих. (markdown, rst, textile ...) Markdown буюу түүнтэй ойролцоо форматаар бичигдсэн бичлэгүүдээсээ юм хайх, унших зэргийг grep, cat зэрэг коммандуудаар хялбархан зохицуулж болно.
  • Бичлэг оруулах, нийтлэх бүх ажлыг коммандын мөрөөс (command line) зохицуулж болно. Энэ бол үнэхээр таатай залхуурал :) (vim, git)
  • Блогоос маань болж учирж буй сэрвэрийн ачааллыг багасгах :P (Мэдээж энэ бол наргиа. Ийм их хандалт баргийн блогт ирэхгүй)
  • Зүгээр л geek зан гаргаж янз бүрийн юм турших. Статик блогтой байх cool ч юм бил үү :) (Мэдээж энэ бол маш чухал шалтгаан)
